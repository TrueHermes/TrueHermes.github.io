本静态博客搭建指北

2020-02-20
  • 教程
    • 博客搭建

前言

过去

我自小就有搭建一个属于自己的网站的想法，记得还在读小学的时候，就百度过建设网站的办法，当时天真地以为搭建网站就是注册一个网址这么简单的事情，现在想想都觉得好笑。但是我真的就在一个不知道是什么的地方“注册”了一个网站，网址是我一时兴起随便取的，但是跳转到的是某个类似于hao123的首页，并不是我想象中的那种具有现代设计美感的网页。于是乎在自己感觉被骗了之后，我选择了打开QQ空间，开始装饰，当时很流行底部花藤，各种样式播放器等等的UI插件，觉得就像在在打造一个属于自己的虚拟的家（羞涩）。

契机

一转眼，现在已经是一名即将本科毕业的大学生了。前不自学过Python，并且用Django+Bootstrap搭建了一个小小的水果市场，是跟着B站上的一个教程bilibili：跟着Mosh学Python，搭配书籍《Python编程：从入门到实践》一起食用，最终效果如图

是小有成就感的，但是这些都是很基础的东西。直到后来又在B站（全中国最大的学习网站）上刷到了一个up主的视频bilibili：手把手教你从0开始搭建自己的个人博客，才真正开始动手开始搭建一个自己的博客。下面便是对搭建本博客的一个教程总结。

说明：本教程是基于Windows操作系统而进行的！

第一步 安装Node.js，Git和hexo

Node.js

  • 在Node.js官网下载长期支持版，然后就是常规的安装，安装过程一直默认点击Next即可。

  • 按下win+r 键打开运行界面，输入cmd打开命令提示符窗口，输入指令node -v和npm -v可以查询版本号

Git

  • 在Git官网中选择64-bit Git for Windows Setup下载安装包

    如果电脑是32位的请选择32-bit版本下载

  • 安装过程可以一直点Next

    如果国内下载速度过慢，或者下载中途失败，可以尝试从其他国内途径寻找资源，也可以采用科学上网暴力下载。

hexo

    这里安装淘宝镜像源的目的是为了后面能够快速安装hexo

  • 淘宝镜像源安装好后也可以用cnpm -v查询版本号等相关信息

  • 输入下面的指令安装hexo框架

  • 最后可以输入hexo -v查询版本号

第二步 配置环境

创建空文件夹

  • 可以利用mkdir blog在命令提示符窗口创建，blog可以根据自己偏好更改

    新建的文件夹在输入行的前缀地址

  • 也可以在文件管理器中直接新建文件夹

在命令提示符窗口中转到blog文件夹

  • 利用cd address等命令跳转至blog文件，address是指blog文件夹的地址

  • 也可以在blog文件夹的地址行输入cmd打开新的窗口。

初始化文件夹

  • 在命令提示符窗口中输入hexo init命令

    如果Windows用户在前面没有安装Git，在这一步就会报错提示’git’指令不存在
    这一步使用国内的网络极有可能会非常非常慢，还有较小概率会出错，建议科学上网

启动博客

  • 输入hexo s指令启动博客

  • 打开浏览器跳转至http://localhost:4000/，即可看到刚搭建好的博客

Hello World

  • 博客自带了一篇《Hello World》的文章，里面有对hexo的基础命令的介绍

  • 利用hexo new "my first blog"指令创建第一篇博客文章

  • 利用Vim，VS Code等来对你的第一篇博客文章“my first blog.md”进行编辑，也可以直接用Notepad来进行编辑，利用markdown语法进行标注，保存

  • 再回到博客页面，刷新一下就可以看到你刚写的文章

第三步 部署到GitHub

注册/登录GitHub

  • 国内浏览GitHub速度慢得可怕，建议科学上网

New repository

  • 在name这一栏填写自己的GitHub ID.github.io

务必保证开头是自己的ID

安装hexo-deployer-git

修改_config.yml文件

  • 可以利用Vim， VS Code或者Notepad修改blog\_config.yml

部署到GitHub仓库

  • 在命令提示符输入hexo d将文件发送到你的仓库

  • 第一次使用这个命令会先让你登录GitHub，然后会提示你告诉他who you are，根据他的提示输入相应的命令设置你的邮箱和ID

    ID一定要是GitHub的ID，输错没关系，再输一遍命令覆盖就行

访问主页

  • 浏览器访问https://github.com/TrueHermes/TrueHermes.github.io.git，即可看到你部署到GitHub的刚搭建好的博客

    同样也要把地址里的ID替换成自己的ID

  • 此时再去刷新刚创建好的仓库，就可以看到部署到GitHub的文件

第四步 美化与优化

搜索主题

  • 去hexo主题网站查找自己喜欢的主题

  • 下载主题的方式有两种

命令提示符下载主题

  • 以我用的主题为例

  • 每个主题的下载指令差别不大，上述的yscoder是主题的作者，indigo是主题的名字，themes\indigo将在你的博客文件夹下的themes文件夹下新建一个叫indigo的文件夹存储你下载的主题的信息

  • 根据你想下载主题的信息，把作者、名字、存储文件夹的名字修改成想下载主题相对应的信息，即可下载，务必将主题存储在themes文件夹下新建的主题文件夹

在作者的GitHub或者其他途径上下载

  • 如果作者在其他社区上传了主题资源，可以去相应的地方下载

  • 将得到的文件解压缩到博客文件夹下themes\ fileName，fileName自己确定，建议同步主题名字

  • 此时返回http://localhost:4000/就可以看到效果

  • 最后利用hexo d部署到GitHub即可
